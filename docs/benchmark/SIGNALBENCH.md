# SignalBench protocol

SignalBench treats every product as an external executable that consumes a frozen JSON request and writes a JSON result. It does not vendor or import competitor source code.

A valid pair has the same task hash, repository tree, prompt, model, reasoning, context window, verifier, permissions, timeout and cache mode. Failed or unverifiable work receives zero credit. Superiority requires at least ten valid pairs, no quality regression, no security regression, no verifier skip, quota measurements and a 95% bootstrap confidence-interval lower bound above one. A 5X claim additionally uses SignalCore's stricter claim-governance gate.
