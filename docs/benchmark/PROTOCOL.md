# Paired benchmark protocol

## Construction versus evidence

A 20X/30X/100X config validates intended workload shape only. Claim-bearing difficulty must be derived from actual observed repository size, graph complexity, output volume, duration, process pressure, verifier count, context volume, history length, session continuity and independent faults.

## Paired identity

Both arms must use the same:

- model and version
- reasoning level
- prompt hash
- repository tree
- workload hash
- verifier hash
- permissions
- timeout
- cache condition

Failed runs, unequal verified work, missing partners and skipped required verifiers receive no efficiency credit.

## 5X gate

A 5X receipt requires:

- at least 10 valid paired repetitions
- actual positive quota telemetry for every pair
- observed qualified difficulty
- zero success-rate regression
- zero required-verifier skips
- zero security, stale-evidence or recovery regressions
- median ratio ≥ 5.00X
- geometric mean ratio ≥ 5.00X
- 95% bootstrap confidence-interval lower bound ≥ 5.00X

Token estimates, synthetic quota values and configured difficulty cannot pass this gate.
