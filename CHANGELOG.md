# Changelog

## 0.0.1 — Current pre-release

Syntavra remains version-locked to **0.0.1 / pre-release** until the repository owner explicitly authorizes a version or channel change. Architectural components do not carry independent release numbers.

### Unified product scope

- Consolidated the repository under the Syntavra identity, canonical `syntavra_runtime` package, `syntavra` executable and `setup / status / run / prove` product model.
- Added a one-command, backup-first installation surface with host detection, configuration planning, doctor checks and rollback-capable operations.
- Added provider, framework, CLI, IDE, extension, SDK, MCP and remote-workspace adapter contracts with explicit maturity and external-certification boundaries.
- Added Semantic Intelligence with incremental repository parsing, symbol navigation, semantic index import, runtime evidence, impact analysis and affected-test traversal.
- Added SignalBench with repository-task planning, paired schedules, identical-arm controls, provider receipt requirements and fail-closed superiority gates.
- Added bounded active context over exact external history, typed context compilation, output externalization, multi-view session memory and deterministic recovery handles.
- Added a bounded autonomous coding-agent loop with isolated workspaces, explicit authorization, verifier execution, anti-loop controls and exact rollback receipts.
- Added capability security, secretless provider transport, probed native sandbox backends and honest fail-closed reporting when enforcement is unavailable.
- Added headless queue execution, resumable job bundles, interactive token and cost visibility, fault-injection campaigns, atomic updates and automatic rollback.
- Added signed artifact, SBOM, provenance, reproducibility, migration, recovery and public-evidence infrastructure.
- Locked Python, TypeScript, skill, marketplace, extension, CodeMeta, CLI and validation identity to 0.0.1.
